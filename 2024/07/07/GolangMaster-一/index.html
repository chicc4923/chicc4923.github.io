Golang Master Lecture Series 1

后端开发教程，技术栈：

Golang-Docker-Postgres

本课程主要通过上述技术栈对一个完整的后端项目进行设计、开发并部署。

构建一个简单的银行系统

这个简单的银行系统将为前端提供 API，并执行以下操作：

  1. Create and manage account
    owner,balance,currency
  2. Record all balance changes
    Create an account entry for each change
  3. Money transfer transaction
    Perform money transfer between 2 accounts consistently within a transcation

1. 数据库设计

学习目标：

  1. 如何使用 dbdiagram.io 设计 SQL 数据库架构。
  2. 然后将架构另存为 png 或者 pdf 图表。
  3. 生成 sql 代码在选择的数据库引擎中创建架构。比如 mysql，postgres，sqlserver

1.1 从 dbdiagram.io 开始

首先设计账户表（accounts）
这个表将记录用户账户的创建时间、拥有者，货币名称，余额。

然后是条目表（entries）：
这个表将记录账户余额的所有更改，所以需要一个自增 ID，与账户表 id 关联的外键，创建时间和更改数目，这个更改数目可以是正数，也可以是负数

最后一张表为转账表（transfers）
这张表只记录发生在银行内部的转账，需要一个自增主键，来款账户和去款账户以及转账金额和创建时间

然后可以直接导出为 postgres sql 语句或 mysql 语句

其中 001 是版本号，后面是文件名，sql 就是设置的后缀，还有一个是 up，一个是 down。
分别生成 up 和 down 是编写数据库时的最佳实践，运行 up 文件来对数据库进行前向更改，如果需要回滚，就运行 down 文件。
所以当我们运行 migrate up 时，就会执行这个文件目录下的 up 文件，然后按照前缀的版本号依次运行，反之亦然。
然后将章节一中设计的 sql 文件复制到 up 文件内。而 down 应该回滚 up 文件中所作的更改，所以需要删除新建的表
